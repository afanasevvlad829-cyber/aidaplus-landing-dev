# Agent Rules

## General
Работай осторожно и по шагам.

## Перед изменением кода
1. Прочитать AI_CONTEXT.md
2. Смотреть только релевантные файлы
3. Сначала кратко объяснить план
4. Менять минимально возможный код

## Нельзя без запроса
- переписывать весь лендинг
- менять бизнес-логику
- подключать новые фреймворки
- переименовывать много файлов
- менять структуру проекта

## UI
- избегать поломки верстки
- не менять визуальную иерархию
- править CSS точечно

## JS
- выносить логику в модули
- не дублировать состояние
- использовать понятные ключи localStorage

## Build
Итоговая версия должна собираться в один HTML файл для Tilda
