# Audit Package Notes

## 1) Состав текущей сборки (runtime)
- index.html
- src/styles/main.css
- src/scripts/main.js
- assets/ (иконки и логотип)

Этот пакет не содержит черновые каталоги, бэкапы, временные артефакты, test/CI-файлы и альтернативные архитектурные ветки.

## 2) Технические моменты (для аудита)
- Архитектура UI: один state-источник в main.js, детерминированный рендер.
- Переключение режимов: compact/full через setMode + localStorage.
- Hero Funnel: шаги, CTA и overlay-логика управляются состоянием.
- Дизайн-токены и визуальные правки централизованы в main.css.

## 3) DEV/технические механики, НЕ для финального production релиза
- Режим аудита по query-параметру: `?audit=1`.
- Технические audit-панели и нумерация блоков (Hero Audit Map / Audit Actions / Site Stages).
- Перетаскивание audit-бейджей и журнал изменений для внутренней отладки.
- Служебный TECH badge (build/time) и его локальная настройка скрытия.

## 4) Пометка для финальной версии
В финальный production релиз не выгружаются технические audit-механики и вспомогательные dev-интерфейсы.
Пакет подготовлен для функционально-визуального аудита текущей сборки.
