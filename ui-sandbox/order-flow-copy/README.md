# Order Flow Sandbox (isolated copy)

Изолированная sandbox-страница для отработки order flow:
- вход в список смен
- выбор смены
- first offer (72ч)
- final offer (24ч после истечения first)
- финальная форма бронирования
- локальный лог событий (payload preview для Telegram)

## Что важно
- Не использует production-роуты и не меняет текущий `index.html`.
- Все данные/состояние сохраняются в `localStorage` ключе `of:sandbox:v1`.
- Это отдельная копия в `ui-sandbox/`.

## Запуск локально

Из корня проекта:

```bash
python3 -m http.server 4173
```

Открыть:

- `http://127.0.0.1:4173/ui-sandbox/order-flow-copy/`

## Быстрый сценарий проверки

1. Нажать `Выбрать смену и цену`.
2. Открыть смену и нажать `Забронировать смену`.
3. В модалке дождаться расчёта цены.
4. Вариант A: ввести телефон + согласие -> `Зафиксировать ... и продолжить бронь`.
5. Вариант B: `Сохранить только в браузере`.
6. Для теста финального оффера нажать `sandbox: завершить текущий offer`, затем `Получить финальную цену`.
7. Пройти `Подтвердить бронь`.

## Telegram интеграция (опционально)

Можно передать webhook до загрузки `app.js`:

```html
<script>
  window.ORDER_FLOW_CONFIG = {
    telegramWebhookUrl: "https://example.com/webhook"
  };
</script>
```

Если webhook не задан, события остаются только в локальном Event log.
